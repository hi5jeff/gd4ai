## 这是什么

**gd4.ai 是一个「需求驱动」的 AI 工具生态推荐引擎。**

你用大白话描述想做什么 —— 比如「我要做一个电商网站」「生成香水广告的提示词」「给我一个会做 UI 设计的 AI 角色」—— 它直接告诉你**该用哪些 AI 技能(Skill)、MCP 服务器、插件、专家角色和提示词**，并给出**怎么装、怎么配、第一步怎么做**。

一句话：**别人给你一堆链接，我们给你一套能直接照着跑通的方案。**

---

## 我们不是搜索引擎

搜索引擎（谷歌 / 必应）做的是：把全网网页**索引 + 排序**，丢给你一串链接——**东西真假、好坏、能不能用、怎么用，全靠你自己去点、去读、去试、去拼。** 它给的是「信息」，不是「答案」；给的是「线索」，不是「方案」。

gd4.ai 做的是**相反**的事：你说需求，我们从**审核过的库**里选出真能用的东西，**组装成一套能照着做的方案**交给你。

> 一句话：**搜索引擎给你一堆「可能有用的链接」，gd4.ai 给你一套「拿来就能跑的方案」。**

| | 搜索引擎（谷歌/必应） | gd4.ai |
|---|---|---|
| 本质 | 索引全网 + 按相关度排序 | 审核建库 + 按需求组装方案 |
| 你得先知道工具叫什么 | ✅ 要 | ❌ 不用，只说你想干什么 |
| 给你的是 | 一串链接，自己去翻去试 | 能照做的方案（装 / 配 / 第一步） |
| 搜一个提示词 | 指向某个仓库首页 | 直接给**可一键复制的那条** |
| 内容真假好坏 | 全网都有，得自己甄别 | **AI 双重把关**，只推审核过的 |
| 库里没有时 | 硬塞几个不相关的给你 | 老实说「没有」，绝不编造 |
| 覆盖到哪一层 | 「工具名 + 简介」 | 「怎么用、对谁有帮助、第一步」 |

## 我们的特点

### ⭐ 核心：AI 双重把关 —— 进得来的都被审核过，推得出的都在库里

这是 gd4.ai 和「让 AI 直接凭记忆瞎答」的根本区别。AI 在两端各把一道关：

- **入库靠 AI 审核收集**：每一个资源（无论我们自己收的，还是用户提交的），都要先过 AI 审核、读懂、拆解才能进库——判断它*是什么、是否有价值、怎么用、对谁有帮助*；看不懂用法、重复、纯广告的，挡在门外。
- **出库只能 AI 筛选推荐**：给你出方案时，AI **只能从这个审核过的库里筛选和组合**，不能凭空生成、不能引用库外内容，输出前还逐条校验。

> 结果：你看到的每一条推荐，都**真实存在、被我们核实过、且能照着做**。库里没有，就老实告诉你没有——绝不编一个看似合理实则跑不通的答案。

### 1 · 需求 → 整套方案，不是关键词搜索
问「我要做网站 UI 设计」，直接给出 **UI UX Pro Max（技能）+ UI/UX 设计师（角色）+ Figma**，每个都说明为什么需要、怎么用——而不是让你在一长串结果里自己拼。

### 2 · 给「能照做」的内容，不是链接堆
问「香水广告提示词」，返回的是**可复制的提示词本身**，不是甩你一个 GitHub 仓库让你自己进去找。

### 3 · AI 读懂每个资源，写成能上手的说明
入库审核时，AI 基于资源的真实文档（README / 官网）理解它怎么用，且严格 grounded、绝不凭空编使用方法——所以推荐里的「怎么装、第一步怎么做」是真的、能照做的。

### 4 · 两层粒度，粗细都覆盖
- **组件层**：一个工具 / 技能 / 库 = 一条（附安装配置）
- **提示词层**：一条条提示词单独入库、可检索、可复制
> 所以「香水广告提示词」能精确捞出那一条，而不是一个提示词库的链接。

### 5 · 多语言
用任意语言提问，按你的语言给答案（数据统一存储，输出时由 AI 翻译成你提问的语言）。

### 6 · 从小白到专家
新手拿到手把手的步骤和第一条命令；专家直接拿到那个具体组件。

### 7 · 持续扩充，人人可荐
发现好的 GitHub 仓库 / 提示词库 / 工具网站？贴个网址提交，AI 审核理解后拆解入库——库会越长越厚。

## 它是怎么做到的

```
需求  →  混合检索(关键词 + 语义向量)  →  取回候选  →  闭世界 LLM 编排  →  校验去伪  →  可执行方案
```

- **结构化数据集**：统一 schema 的组件库 + 提示词库
- **混合检索**：Meilisearch（关键词）+ pgvector（语义向量），RRF 融合排序
- **闭世界编排**：LLM 只能从检索候选里选、输出前校验，绝不引用库外内容
- **向量与推理**：本地 BGE-M3 向量模型 + DeepSeek 级 LLM（在线推理 / 离线加工）
- **前后端**：FastAPI 后端 + 轻量静态前端

数据来源：自建种子 + 从 GitHub 仓库 / 网页 / 提示词库自动理解入库 + 用户提交（审核）。真相以数据库为准，库可重建。
