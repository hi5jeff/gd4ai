# gd4.ai 首页 PRD（产品需求文档）

> 版本 v1.0 · 2026-07-17 · 负责人：gd4.ai
> 范围：公开落地页 `https://gd4.ai/`（`web/index.html` + `web/i18n.js`）。管理后台 `admin.html` 不在本文范围。

---

## 1. 背景与定位

gd4.ai 是一个**需求驱动的 AI 工具生态推荐引擎**：用户用大白话描述想做的事，系统从人工+AI 审核过的库里给出「该用哪些技能/MCP/插件/角色/提示词 + 怎么装/配/上手」的可执行方案。

首页是**唯一入口和核心界面**——它既是营销落地页（讲清「我们是什么、凭什么不一样」），又是产品本身（一个搜索框即完成推荐）。二者合一，不做多页跳转。

**一句话定位**：别人给你一堆链接，我们给你一套能直接跑通的方案。

---

## 2. 目标与非目标

### 目标
- **G1 · 3 秒讲清价值**：新访客进入 3 秒内理解「说需求 → 得方案」，且明白这不是搜索引擎。
- **G2 · 零门槛发起**：不需要注册/引导，打开即可在搜索框提问。
- **G3 · 结果可直接用**：推荐结果包含可复制命令/提示词、怎么用、来源，用户能照做。
- **G4 · 建立信任**：通过「人工核实 / AI 双重把关 / 库存规模」传达可信度，降低「AI 瞎编」疑虑。
- **G5 · 可增长**：用户能提交资源；内容/品牌可被搜索引擎与社交平台正确收录传播。

### 非目标
- ❌ 不做用户账号/登录/个人中心（首页阶段）。
- ❌ 不做完整的目录浏览/分类页（我们不是导航站）。
- ❌ 不做多页站点（关于/定价/博客等后续再说）。

## 5. 模块详述

### 5.1 顶栏
- **品牌**：`gd4.ai`，`.` 为琥珀强调色。点击回到首页（清空结果）。
- **语言切换**：`中文 / EN`，当前语言高亮。切换即时生效（不刷新），记忆到 `localStorage`。
- 需求：轻量、不喧宾夺主；移动端保持单行。

### 5.2 英雄区（首屏，核心）
| 元素 | 内容 | 要求 |
|---|---|---|
| eyebrow | "精选 AI 工具生态" | 小字、字距拉开、呼吸点动画 |
| 大标题 | "说出你想做的事，得到一套能直接跑通的 AI 工具方案" | 衬线（Instrument Serif/宋体），左对齐，是页面视觉重心 |
| 搜索框 | 占位示例 + 主按钮"推荐方案" | 高度醒目；聚焦有琥珀光环；回车即提交；内嵌搜索图标；按钮带箭头 |
| 快捷芯片 | 6 个（电商/人像提示词/省Token/游戏动画/平面设计/室内装修） | 点击即以该需求发起搜索；lucide 图标；hover 上浮 |
| 实时库存条 | `1,462 已验证资源 · 448 Skill · 393 Tool · 289 Role · … · 13,225 提示词` | 数据来自 `/api/stats`（缓存，不实时压库）；填补空旷、增强可信 |

- **入场动效**：eyebrow → 标题 → 搜索框 → 芯片 → 库存条，依次上浮淡入（错峰）。
- **约束**：`prefers-reduced-motion` 时关闭动效。

### 5.3 结果区
提问后在英雄区下方渲染，卡片**错峰浮现**。三种结果形态：

- **回答概述**（answer）：一段话方案总述；空结果时为警示样式。
- **方案包（playbook）**：完整工作流 —— 标题 + 概述 + 需要的组件（卡片）+ 操作步骤（编号）+ 起手提示词 + 常见坑。
- **组件卡（component）**：类型徽章 + 名称 + ⭐/难度 + 推荐理由 + 描述 + **安装命令（可复制）** + 用法说明 + **第一步提示词（可复制）** + 需要准备 + 安全提示 + 来源链接。
- **提示词卡（prompt）**：标题 + 摘要 + 场景/风格标签 + **提示词正文（一键复制）**。
- **空结果兜底**：库里没有精确匹配时，明确告知"暂无"，并（若有）给出最接近内容作参考——**绝不硬凑/编造**。

**关键交互**：所有命令/提示词均有「复制」按钮，复制后短暂反馈"已复制"。

### 5.4 提交资源弹窗
- 入口：页脚「推荐资源」。
- 内容：标题 + 说明 + URL 输入 + 备注输入 + 提交/关闭。
- 逻辑：POST `/api/submit`；成功/重复/失败分别反馈；成功后 1.6s 自动关闭。
- 价值话术：贴网址 → 我们审核后导入库里。

### 5.5 页脚
- 一句话说明（所有推荐来自人工核实的库）+ 「推荐资源」链接（lucide 加号）。

## 6. 关键交互流程

1. **搜索**：输入/点芯片 → 载入态（旋转 + 文案）→ 调 `/api/recommend?lang=` → 渲染结果（错峰浮现）→ 失败显示错误态。
2. **复制**：点命令/提示词旁「复制」→ 写入剪贴板 → 按钮变"已复制✓" 1.5s 后还原。
3. **切语言**：点 中文/EN → 全站文案即时切换；已出结果保留（LLM 已按语言输出）。
4. **提交资源**：页脚入口 → 弹窗 → 提交 → 结果反馈。
5. **深链**：`gd4.ai/?q=<需求>` 打开即自动发起该搜索（可分享、利于 SEO）。

## 8. 非功能需求

- **性能**：首屏静态直出（英雄区文本内嵌 HTML，不依赖 JS 才可见）；库存条异步加载；字体 `display=swap` 防阻塞。
- **SEO**：完整 meta / Open Graph / Twitter Card / JSON-LD（WebSite+SearchAction、WebApplication）/ canonical / hreflang / robots / sitemap / og 封面图 / noscript 兜底内容。
- **多语言**：zh/en，`localStorage` 记忆 + 浏览器语言探测；结果语言随提问语言。
- **响应式**：≥560px 桌面，<560px 移动端（按钮收窄、间距压缩）。
- **无障碍/降级**：尊重 `prefers-reduced-motion`；无 JS 时 noscript 展示核心说明。
- **兼容**：任意现代浏览器；纯静态（HTML+JS），无构建步骤。

## 9. 数据与接口依赖

| 接口 | 用途 | 备注 |

| `POST /api/recommend?lang=` | 需求 → 推荐结果 | 闭世界编排；Redis 缓存 |

| `GET /api/health` | 健康探测 | 备用 |

## 10. 设计规范（反「AI 味」）

- **配色**：近黑暖调背景 + 发丝级边框 + **单一琥珀强调色**（`#e7ad5f`）；徽章低饱和、按类型描边，不做彩虹。
- **排版**：Inter 正文 + Instrument Serif / 宋体大标题（衬线×无衬线混排是核心反 slop 信号）；左对齐编辑式版式，不全居中。
- **图标**：一律 lucide 线性图标（内联 SVG），**禁用 emoji**。
- **动效**：氛围光晕缓慢漂移 + 入场/结果错峰浮现 + 克制微交互；不炫技。
- **文案**：口语、具体、带对比（"我们不是搜索引擎"）；禁用 em-dash 堆砌与空话套话。

## 11. 成功指标（建议埋点）

- **激活率**：进站到发起首次搜索的比例（目标锚点，越高越好）。
- **有效结果率**：返回非空结果 / 总搜索（衡量库覆盖）。
- **复制率**：复制命令/提示词的会话占比（衡量"可用性"是否真被使用）。
- **空结果词**：记录查空的需求（`gap_log`），驱动补库。
- **提交量**：用户提交资源数。
- **分享/收录**：来自 `?q=` 深链与搜索引擎的访问量。
